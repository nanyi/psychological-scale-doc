## 2. 概述

### 2.1 模块定位

报告生成与导出模块是心理测评系统的核心服务模块，负责将测评结果转化为结构化的专业报告，并支持多格式导出（Word、PDF），满足用户存档、打印、分发等需求。

### 2.2 功能摘要

本模块实现测评报告的自动生成、报告模板管理、多格式导出、报告查看与下载等功能，支持内置量表的自定义报告和第三方量表API返回报告的统一个人中心管理。

## 3. 业务需求

### 3.1 功能需求

#### FR-001: 报告自动生成

**功能描述**: 用户完成测评后自动生成测评报告

**输入**:
- 测评任务ID
- 答题记录
- 评分结果

**处理逻辑**:
1. 获取测评任务和答题记录
2. 根据评分规则计算维度得分和总分
3. 根据常模数据判定结果等级
4. 替换报告模板变量生成报告内容
5. 保存报告数据
6. 发送报告生成通知给用户

**输出**: 测评报告

**业务规则**:
- 测评完成后立即生成报告
- 报告生成失败保留重试机制
- 报告数据长期保存

**报告数据结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 报告ID |
| report_no | VARCHAR(32) | 报告编号 |
| task_id | BIGINT | 测评任务ID |
| user_id | BIGINT | 用户ID |
| scale_id | BIGINT | 量表ID |
| scale_name | VARCHAR(200) | 量表名称 |
| total_score | DECIMAL(10,2) | 总分 |
| dimension_scores | TEXT | 维度得分(JSON) |
| result_level | VARCHAR(50) | 结果等级 |
| conclusion | TEXT | 结论 |
| suggestions | TEXT | 建议(JSON数组) |
| report_content | TEXT | 完整报告内容(JSON) |
| source_type | TINYINT | 来源：1-本地生成，2-第三方API |
| third_party_report | TEXT | 第三方报告原始数据 |
| status | TINYINT | 状态：0-生成中，1-已生成，2-生成失败 |
| generate_time | DATETIME | 生成时间 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

#### FR-002: 报告模板管理

**功能描述**: 管理员可配置不同量表的报告模板

**输入**:
- 量表ID
- 模板名称
- 模板内容（HTML/富文本）
- 变量占位符

**处理逻辑**:
1. 创建报告模板
2. 定义模板变量（总分、维度得分、等级等）
3. 设计模板样式
4. 配置变量替换规则
5. 预览模板效果
6. 发布模板

**输出**: 模板配置结果

**业务规则**:
- 一个量表可有多个模板
- 模板支持条件渲染
- 模板样式与导出格式匹配

**报告模板数据结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 模板ID |
| scale_id | BIGINT | 量表ID |
| template_name | VARCHAR(100) | 模板名称 |
| template_type | TINYINT | 模板类型：1-简版，2-详版，3-专业版 |
| template_content | TEXT | 模板内容(HTML) |
| variables | VARCHAR(1000) | 变量列表(JSON) |
| status | TINYINT | 状态：0-草稿，1-已发布 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

**支持的模板变量**:

| 变量名 | 说明 | 示例 |
|--------|------|------|
| $USER_NAME$ | 用户姓名 | 张三 |
| $SCALE_NAME$ | 量表名称 | SCL-90症状自评量表 |
| $TOTAL_SCORE$ | 总分 | 145 |
| $RESULT_LEVEL$ | 结果等级 | 轻度异常 |
| $ASSESS_DATE$ | 测评日期 | 2026-03-09 |
| $DIM_XXX_SCORE$ | 维度得分 | 维度名_得分 |
| $DIM_XXX_LEVEL$ | 维度等级 | 躯体化_正常 |
| $CONCLUSION$ | 综合结论 | 您的测评结果显示... |
| $SUGGESTION_1$ | 建议1 | 建议保持规律作息 |

#### FR-003: 报告查看

**功能描述**: 用户可在个人中心查看测评报告

**输入**:
- 用户ID
- 报告ID或任务ID

**处理逻辑**:
1. 校验用户权限
2. 查询报告数据
3. 渲染报告内容（HTML）
4. 返回报告详情

**输出**: 报告详情

**业务规则**:
- 用户只能查看自己的报告
- 报告查看记录日志
- 支持分享报告（生成临时链接）

#### FR-004: Word文档导出

**功能描述**: 用户可将报告导出为Word文档

**输入**:
- 报告ID
- 模板ID（可选）

**处理逻辑**:
1. 获取报告数据
2. 选择导出模板
3. 组装Word文档内容
4. 插入标题、用户信息、得分详情、结论建议
5. 应用样式和格式
6. 生成Word文件
7. 返回文件下载链接

**输出**: Word文件下载链接

**业务规则**:
- Word文档包含完整报告信息
- 支持自定义水印
- 文件名格式：{用户姓名}_{量表名}_报告_{日期}.docx

**Word文档结构**:

```
{量表名称}测评报告
================================

【基本信息】
姓名：{姓名}
测评日期：{日期}
测评用时：{时长}分钟

【测评结果】
总分：{总分}
结果等级：{等级}

【维度分析】
{维度1}：{得分}（{等级}）
{维度2}：{得分}（{等级}）
...

【综合结论】
{结论内容}

【建议指导】
1. {建议1}
2. {建议2}
...

【报告说明】
本报告仅供参考，不作为诊断依据。
```

#### FR-005: PDF文档导出

**功能描述**: 用户可将报告导出为PDF文档

**输入**:
- 报告ID
- 模板ID（可选）

**处理逻辑**:
1. 获取报告数据
2. 选择导出模板
3. 渲染HTML内容
4. 转换为PDF格式
5. 添加页眉页脚
6. 添加水印（可选）
7. 生成PDF文件
8. 返回文件下载链接

**输出**: PDF文件下载链接

**业务规则**:
- PDF文档与Word内容一致
- 支持A4竖版/横版
- 文件名格式：{用户姓名}_{量表名}_报告_{日期}.pdf

#### FR-006: 第三方报告集成

**功能描述**: 第三方量表平台的报告结果统一在本系统展示和导出

**输入**:
- 第三方回调的报告数据

**处理逻辑**:
1. 接收第三方报告回调
2. 解析报告数据结构
3. 标准化报告格式
4. 保存报告原始数据
5. 生成标准化报告内容
6. 关联到测评任务

**输出**: 集成后的报告

**业务规则**:
- 第三方报告统一存储
- 支持导出为Word/PDF
- 保留第三方原始报告数据

#### 3.3.2 生成报告接口

**路径**: POST /api/report/generate

**请求**:
```json
{
  "taskId": "BIGINT|测评任务ID",
  "templateId": "BIGINT|模板ID(可选)"
}
```

**响应**:
```json
{
  "code": 200,
  "data": {
    "reportId": "BIGINT|报告ID",
    "reportNo": "STRING|报告编号",
    "status": "INT|状态"
  }
}
```

#### 3.3.3 报告详情接口

**路径**: GET /api/report/detail

**请求**:
```

  reportId: BIGINT|报告ID

```

**响应**:
```json
{
  "code": 200,
  "data": {
    "reportId": "BIGINT|报告ID",
    "scaleName": "STRING|量表名称",
    "totalScore": "DECIMAL|总分",
```

## 4. 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 报告生成 | ≤3s | 本地报告生成 |
| Word导出 | ≤5s | Word文件生成 |
| PDF导出 | ≤8s | PDF文件生成 |
| 报告查询 | ≤200ms | 报告列表和详情 |
| 并发导出 | ≥50TPS | 支持并发导出 |

## 5. 可用性需求

- **SLA要求**: 导出接口可用性≥99.5%
- **异步导出**: 大报告支持异步生成
- **文件存储**: 导出文件保留7天
- **失败重试**: 导出失败自动重试3次

## 6. 安全性需求

- **权限校验**: 只能导出自己的报告
- **下载链接**: 一次性有效或限时有效
- **敏感信息**: 报告脱敏处理
- **审计日志**: 导出操作记录日志
