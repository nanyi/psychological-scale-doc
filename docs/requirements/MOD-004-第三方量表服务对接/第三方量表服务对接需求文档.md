## 2. 概述

### 2.1 模块定位

第三方量表服务对接模块是心理测评系统的核心集成模块，负责与第三方量表服务平台进行API对接，实现量表题目获取、答题数据提交、报告结果回调等全流程集成。

### 2.2 功能摘要

本模块实现第三方量表平台的注册与配置、量表题目同步、答题过程管理、报告结果回调等完整对接流程，支持对接多个第三方量表平台，统一管理第三方量表与内置量表。

## 3. 业务需求

### 3.1 功能需求

#### FR-001: 第三方平台配置

**功能描述**: 管理员可配置第三方量表服务平台的API对接信息

**输入**:
- 平台名称
- API基础地址
- AppKey/AppSecret
- 回调地址
- 同步策略

**处理逻辑**:
1. 管理员在后台添加第三方平台配置
2. 系统校验API连接性（调用测试接口）
3. 保存配置信息（密钥加密存储）
4. 启用/停用第三方平台

**输出**: 平台配置结果

**业务规则**:
- AppSecret必须加密存储
- 支持配置多个第三方平台
- 平台停用后不影响已有数据

**第三方平台配置数据结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 平台ID |
| platform_name | VARCHAR(100) | 平台名称 |
| api_base_url | VARCHAR(200) | API基础地址 |
| app_key | VARCHAR(100) | AppKey |
| app_secret | VARCHAR(500) | AppSecret（加密） |
| callback_url | VARCHAR(200) | 回调地址 |
| sync_strategy | TINYINT | 同步策略：1-实时，2-定时 |
| status | TINYINT | 状态：0-停用，1-启用 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

#### FR-002: 量表同步

**功能描述**: 从第三方平台同步量表信息到本系统

**输入**:
- 第三方平台ID
- 量表ID列表（可选，为空则同步全部）

**处理逻辑**:
1. 调用第三方平台API获取量表列表
2. 解析量表基本信息（名称、描述、适用人群）
3. 同步量表题目和选项
4. 保存到本地量表数据库
5. 建立本地量表与第三方量表的映射关系

**输出**: 同步结果（成功数量、失败数量、错误详情）

**业务规则**:
- 支持增量同步和全量同步
- 同步冲突时以第三方平台为准
- 同步失败不影响已存在数据

**量表映射数据结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 映射ID |
| local_scale_id | BIGINT | 本地量表ID |
| third_party_id | VARCHAR(100) | 第三方量表ID |
| platform_id | BIGINT | 第三方平台ID |
| create_time | DATETIME | 创建时间 |

#### FR-003: 题目获取

**功能描述**: 用户开始测评时，从第三方平台获取量表题目

**输入**:
- 本地量表ID
- 用户ID

**处理逻辑**:
1. 查询量表映射，获取第三方平台信息
2. 调用第三方平台API获取题目
3. 缓存题目信息（Redis，TTL=30分钟）
4. 返回题目给前端

**输出**: 量表题目列表

**业务规则**:
- 题目必须在30分钟内提交
- 题目缓存后不支持修改
- 第三方平台故障时返回缓存题目

**第三方题目获取请求示例**:
```
GET /api/thirdparty/scale/{thirdPartyScaleId}/questions
Headers:
  X-App-Key: xxxxxx
  X-Timestamp: 1234567890
  X-Signature: xxxxxx
```

**第三方题目获取响应示例**:
```json
{
  "code": 200,
  "data": {
    "scaleId": "TSP001",
    "scaleName": "SCL-90症状自评量表",
    "questions": [
      {
        "questionId": "Q1",
        "content": "头痛",
        "type": "single",
        "options": [
          {"value": "1", "text": "从无"},
          {"value": "2", "text": "轻度"},
          {"value": "3", "text": "中度"},
          {"value": "4", "text": "偏重"},
          {"value": "5", "text": "严重"}
        ]
      }
    ],
    "expireTime": 1800
  }
}
```

#### FR-004: 答题数据提交

**功能描述**: 用户完成答题后，提交答案到第三方平台进行评分

**输入**:
- 本地量表ID
- 用户ID
- 答题记录（问题ID、选项值）

**处理逻辑**:
1. 校验答题完整性
2. 加密签名组装请求参数
3. 调用第三方平台API提交答案
4. 获取评分任务ID
5. 创建本地答题记录
6. 异步等待第三方回调

**输出**: 提交结果、任务ID

**业务规则**:
- 答题未完整不允许提交
- 提交后答案不可修改
- 第三方响应超时自动重试3次

**答题数据提交请求示例**:
```
POST /api/thirdparty/scale/{thirdPartyScaleId}/submit
Headers:
  X-App-Key: xxxxxx
  X-Timestamp: 1234567890
  X-Signature: xxxxxx
Body:
{
  "userId": "user_001",
  "answers": [
    {"questionId": "Q1", "value": "3"},
    {"questionId": "Q2", "value": "2"}
  ],
  "submitTime": "2026-03-09 10:00:00"
}
```

**答题数据提交响应示例**:
```json
{
  "code": 200,
  "data": {
    "taskId": "TASK202603090001",
    "status": "processing",
    "estimatedTime": 5
  }
}
```

#### FR-005: 报告回调

**功能描述**: 第三方平台完成评分后，异步回调本系统获取报告结果

**输入**:
- 任务ID
- 报告状态
- 报告数据（维度得分、总分、结论等）

**处理逻辑**:
1. 验证回调签名
2. 根据任务ID查询答题记录
3. 更新答题记录状态
4. 保存报告结果
5. 发送通知给用户（测评完成）

**输出**: 回调处理结果

**业务规则**:
- 签名验证失败拒绝处理
- 回调必须幂等处理
- 报告数据加密存储

**报告回调请求示例**:
```
POST /api/thirdparty/callback/report
Body:
{
  "taskId": "TASK202603090001",
  "status": "completed",
  "reportData": {
    "totalScore": 145,
    "dimensions": [
      {
        "name": "躯体化",
        "score": 12,
        "level": "正常"
      }
    ],
    "conclusion": "您的SCL-90测评结果提示...",
    "suggestions": ["建议保持规律作息", "建议适当运动"]
  },
  "completedTime": "2026-03-09 10:05:00"
}
```

**报告回调响应示例**:
```json
{
  "code": 200,
  "message": "success"
}
```

#### FR-006: 第三方量表查询

**功能描述**: 用户可浏览和搜索第三方平台同步过来的量表

**输入**:
- 关键词
- 分类
- 分页信息

**处理逻辑**:
1. 查询已同步的第三方量表
2. 过滤已下架量表
3. 返回量表列表（含基本信息）

**输出**: 第三方量表列表

**业务规则**:
- 只显示已启用的量表
- 搜索支持量表名称和描述

## 4. 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 题目获取 | ≤1s | 从第三方API获取题目 |
| 答案提交 | ≤500ms | 提交答案到第三方 |
| 回调处理 | ≤200ms | 处理第三方回调 |
| 并发答题 | ≥200TPS | 支持200个并发答题 |
| 缓存命中率 | ≥90% | 题目缓存命中率 |

## 5. 可用性需求

- **SLA要求**: 第三方接口可用性≥99.5%
- **降级策略**: 第三方平台故障时，使用本地缓存题目，标记报告生成延迟
- **重试机制**: 提交失败自动重试3次，间隔指数增长
- **熔断保护**: 连续失败10次触发熔断，30分钟后恢复

## 6. 安全性需求

- **签名验证**: 所有第三方回调必须验证签名
- **密钥管理**: AppSecret加密存储，定期轮换
- **请求验签**: 发送给第三方的请求必须签名
- **IP白名单**: 仅允许第三方平台IP回调
- **数据加密**: 报告数据加密存储
