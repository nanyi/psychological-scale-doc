## 2. 概述

### 2.1 模块定位

量表库管理与测评执行模块是心理测评系统的基础功能模块，负责量表的创建、配置、管理，以及用户测评过程的执行，是系统的核心业务模块。

### 2.2 功能摘要

本模块实现量表的完整生命周期管理，包括量表基本信息维护、维度因子设计、题目选项配置、评分算法定义、测评执行控制等功能，同时支持内置量表、第三方量表、自定义量表的统一管理。

## 3. 业务需求

### 3.1 功能需求

#### FR-001: 量表基础信息管理

**功能描述**: 管理员可创建、编辑、启用/停用、删除量表

**输入**:
- 量表名称、英文名
- 量表分类（心理健康、人格测评、职业测评、情绪测评等）
- 适用人群
- 量表描述
- 指导语
- 预计测评时长
- 题目数量

**处理逻辑**:
1. 管理员创建量表基本信息
2. 系统生成量表唯一编码
3. 配置量表维度因子结构
4. 添加题目和选项
5. 配置评分规则
6. 设置完成后发布量表

**输出**: 量表基本信息

**业务规则**:
- 量表编码全局唯一
- 删除量表为逻辑删除
- 停用量表后不可购买和测评

**量表基础信息数据结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 量表ID |
| scale_code | VARCHAR(32) | 量表编码（唯一） |
| scale_name | VARCHAR(200) | 量表名称 |
| scale_name_en | VARCHAR(200) | 英文名 |
| category | VARCHAR(50) | 分类 |
| target_audience | VARCHAR(200) | 适用人群 |
| description | TEXT | 描述 |
| instruction | TEXT | 指导语 |
| duration | INT | 预计时长（分钟） |
| question_count | INT | 题目数量 |
| dimension_count | INT | 维度数量 |
| source_type | TINYINT | 来源：1-内置，2-第三方，3-自定义 |
| third_party_id | VARCHAR(100) | 第三方ID |
| price | DECIMAL(10,2) | 价格 |
| status | TINYINT | 状态：0-草稿，1-已发布，2-已下架 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| deleted | TINYINT | 逻辑删除 |

#### FR-002: 量表维度因子管理

**功能描述**: 配置量表的维度、因子及其评分权重

**输入**:
- 维度名称
- 因子名称
- 计算公式
- 权重
- 常模数据

**处理逻辑**:
1. 添加维度定义
2. 添加因子定义
3. 配置维度与因子的包含关系
4. 设置评分计算公式
5. 导入常模数据（年龄、性别分组）

**输出**: 维度因子结构

**业务规则**:
- 一个量表可有多个维度
- 一个维度可有多个因子
- 评分公式支持四则运算和函数

**维度数据结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 维度ID |
| scale_id | BIGINT | 量表ID |
| dimension_code | VARCHAR(32) | 维度编码 |
| dimension_name | VARCHAR(100) | 维度名称 |
| sort_order | INT | 排序 |
| create_time | DATETIME | 创建时间 |

**因子数据结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 因子ID |
| dimension_id | BIGINT | 维度ID |
| factor_code | VARCHAR(32) | 因子编码 |
| factor_name | VARCHAR(100) | 因子名称 |
| formula | VARCHAR(500) | 计算公式 |
| weight | DECIMAL(5,2) | 权重 |
| sort_order | INT | 排序 |
| create_time | DATETIME | 创建时间 |

**常模数据结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 常模ID |
| scale_id | BIGINT | 量表ID |
| dimension_id | BIGINT | 维度ID |
| group_type | VARCHAR(20) | 分组类型（年龄/性别） |

| mean_score | DECIMAL(10,2) | 平均分 |
| std_dev | DECIMAL(10,2) | 标准差 |
| create_time | DATETIME | 创建时间 |

#### FR-003: 题目与选项管理

**功能描述**: 配置量表的题目、选项及计分规则

**输入**:
- 题目内容
- 题目类型（单选、多选、量表题、问答）
- 题目附件（图片/音频）
- 选项列表
- 每个选项的分数
- 题目与维度的关联

**处理逻辑**:
1. 添加题目基本信息
2. 配置题目类型
3. 添加选项及分值
4. 设置题目与维度/因子的映射
5. 设置反向计分题目
6. 预览题目效果

**输出**: 题目列表

**业务规则**:
- 题目编号自动生成
- 选项分值可为负数（反向计分）
- 同一量表题目编号唯一

**题目数据结构**:

|--------|------|------|

| attachment_url | VARCHAR(500) | 附件地址 |
| question_type | TINYINT | 题目类型：1-单选，2-多选，3-量表题，4-问答 |
| is_reverse | TINYINT | 是否反向计分：0-否，1-是 |
| dimension_id | BIGINT | 所属维度 |
| factor_id | BIGINT | 所属因子 |
| sort_order | INT | 排序 |
| is_required | TINYINT | 是否必答：0-否，1-是 |
| create_time | DATETIME | 创建时间 |

**选项数据结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 选项ID |
| question_id | BIGINT | 题目ID |
| option_no | VARCHAR(10) | 选项编号 |
| option_text | VARCHAR(500) | 选项文本 |
| option_value | DECIMAL(10,2) | 选项分值 |
| sort_order | INT | 排序 |
| create_time | DATETIME | 创建时间 |

#### FR-004: 评分算法配置

**功能描述**: 配置量表的评分算法和结果解释规则

**输入**:
- 维度得分计算公式
- 总分计算方式
- 结果等级划分规则
- 结果解释模板

**处理逻辑**:
1. 配置维度得分计算公式（因子分值加权求和）
2. 配置总分计算方式（维度分值加权求和）
3. 配置结果等级划分（常模对比）
4. 配置结果解释模板（变量替换）

**输出**: 评分规则配置

**业务规则**:
- 公式支持变量：$Q1$, $D1$, $F1$等
- 支持函数：SUM, AVG, MAX, MIN
- 等级划分支持多级

**评分规则数据结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 规则ID |
| scale_id | BIGINT | 量表ID |
| rule_type | TINYINT | 规则类型：1-维度分，2-总分，3-等级 |
| rule_code | VARCHAR(32) | 规则编码 |
| rule_name | VARCHAR(100) | 规则名称 |
| formula | VARCHAR(500) | 计算公式 |
| thresholds | VARCHAR(500) | 阈值（JSON数组） |
| interpretation | TEXT | 解释模板 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

#### FR-005: 测评任务创建

**功能描述**: 用户开始测评时创建测评任务

**输入**:
- 用户ID
- 量表ID
- 来源（购买/企业分配/免费）

**处理逻辑**:
1. 校验用户是否有测评权限
2. 创建测评任务记录
3. 获取量表题目（如为第三方量表调用API）
4. 初始化答题进度
5. 返回题目列表给前端

**输出**: 测评任务信息、题目列表

**业务规则**:
- 测评任务有效期30分钟
- 同一量表只能有一个进行中的任务
- 未完成的测评记录保存草稿

**测评任务数据结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 任务ID |
| task_no | VARCHAR(32) | 任务编号 |
| user_id | BIGINT | 用户ID |
| scale_id | BIGINT | 量表ID |
| source_type | TINYINT | 来源：1-购买，2-企业分配，3-免费 |
| source_id | BIGINT | 来源ID（订单ID/配额ID） |
| status | TINYINT | 状态：0-待开始，1-进行中，2-已完成，3-已超时 |
| progress | INT | 进度（当前题目/总题目） |
| start_time | DATETIME | 开始时间 |
| finish_time | DATETIME | 完成时间 |
| expire_time | DATETIME | 过期时间 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

#### FR-006: 答题过程管理

**功能描述**: 管理用户答题过程的记录和控制

**输入**:
- 任务ID
- 题目ID
- 答案内容

**处理逻辑**:
1. 校验任务状态
2. 保存用户答案
3. 更新答题进度
4. 检查是否完成所有题目
5. 如完成，提交测评

**输出**: 保存结果、下一步操作

**业务规则**:
- 必答题未作答不可跳转
- 支持草稿保存
- 答题过程实时存储

**答题记录数据结构**:

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 记录ID |
| task_id | BIGINT | 任务ID |
| question_id | BIGINT | 题目ID |
| answer_value | VARCHAR(500) | 答案值 |
| answer_text | TEXT | 答案文本 |
| score | DECIMAL(10,2) | 得分 |
| answer_time | INT | 答题用时（秒） |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

#### FR-007: 测评提交与完成

**功能描述**: 用户完成答题后提交测评

**输入**:
- 任务ID

**处理逻辑**:
1. 校验答题完整性
2. 计算各维度得分
3. 计算总分
4. 判定结果等级
5. 生成结果解释
6. 更新任务状态
7. 触发报告生成（如为第三方量表）

**输出**: 测评结果

**业务规则**:
- 存在未作答题目不允许提交
- 提交后不可修改答案
- 测评结果长期保存

  "scaleName": "STRING|量表名称",

  "category": "STRING|分类",

## 4. 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 量表查询 | ≤200ms | 量表列表和详情查询 |
| 题目加载 | ≤500ms | 测评题目加载 |
| 答题保存 | ≤100ms | 单题答案保存 |
| 测评提交 | ≤1s | 提交并计算结果 |
| 并发测评 | ≥500TPS | 支持高并发测评 |

## 5. 可用性需求

- **SLA要求**: 测评接口可用性≥99.9%
- **答题保护**: 答题过程每30秒自动保存草稿
- **超时处理**: 超时任务自动标记，保留答题记录
- **断点续答**: 支持从上次答题位置继续

## 6. 安全性需求

- **权限校验**: 测评必须校验用户权限
- **答案加密**: 答题记录加密存储
- **防作弊**: 同一账号同时只能进行一个测评
- **审计日志**: 测评关键操作记录日志
