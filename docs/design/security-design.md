## 3. 身份认证设计

### 3.1 认证流程

```
用户 ──▶ 登录请求 ──▶ 验证凭证 ──▶ 生成JWT ──▶ 返回Token
                              │
                              ▼
                        ┌───────────┐
                        │ 验证码    │
                        │ 校验      │
                        └───────────┘
```

### 3.2 JWT Token设计

| 字段 | 说明 |
|------|------|
| 用户ID | 用户唯一标识 |
| 用户名 | 登录用户名 |
| 角色 | 用户角色列表 |
| 权限 | 用户权限列表 |
| 签发时间 | Token创建时间 |
| 过期时间 | Token过期时间 |

**Token配置**:
- 访问Token有效期：7天
- 刷新Token有效期：30天
- 算法：HS256

### 3.3 登录保护

| 保护措施 | 阈值 | 处理方式 |
|----------|------|----------|
| 密码错误 | 5次 | 锁定30分钟 |
| 验证码错误 | 3次 | 锁定15分钟 |
| 同一IP登录 | 10次/分钟 | 限流 |

## 5. 数据安全设计

### 5.1 数据加密

| 数据类型 | 加密方式 | 密钥管理 |
|----------|----------|----------|
| 用户密码 | BCrypt | 自动 |
| 支付密钥 | AES-256 | 密钥服务 |
| 敏感字段 | AES-256 | 密钥服务 |
| 传输数据 | TLS 1.3 | SSL证书 |

### 5.2 敏感数据脱敏

| 数据类型 | 脱敏规则 | 示例 |
|----------|----------|------|
| 手机号 | 中间4位隐藏 | 138****1234 |
| 身份证号 | 前3后4位 | 110***1234 |
| 银行卡号 | 前6后4位 | 621700***1234 |
| 邮箱 | @前1位 | a***@example.com |

### 5.3 数据备份策略

| 备份类型 | 频率 | 保留时间 |
|----------|------|----------|
| 全量备份 | 每周日 | 90天 |
| 增量备份 | 每日 | 30天 |
| 实时备份 | 实时 | 7天 |
| 异地备份 | 每日 | 180天 |

## 6. 接口安全设计

### 6.1 签名验证

**请求签名算法**:
```
signature = MD5(appKey + timestamp + randomStr + body + appSecret)
```

**签名参数**:
| 参数 | 说明 |
|------|------|
| appKey | 应用标识 |
| timestamp | 时间戳（秒） |
| randomStr | 随机字符串 |
| body | 请求体 |
| appSecret | 应用密钥 |

### 6.2 接口限流

| 限流维度 | 阈值 | 窗口 |
|----------|------|------|
| 单用户 | 100次/分钟 | 按用户ID |
| 单IP | 50次/秒 | 按IP地址 |
| 单接口 | 200次/秒 | 按接口 |

### 6.3 敏感接口

| 接口 | 安全要求 |
|------|----------|
| 登录/注册 | 验证码+限流 |
| 支付 | 签名验证+幂等 |
| 退款 | 二次验证 |
| 导出 | 权限校验+水印 |

## 8. 安全合规标准

### 8.1 等级保护

| 保护等级 | 安全要求 | 适用范围 |
|----------|----------|----------|
| 等保二级 | 基本安全要求 | 普通系统 |
| 等保三级 | 增强安全要求 | 重要系统 |

### 8.2 心理健康数据特别保护

根据《个人信息保护法》和《精神卫生法》要求：

| 数据类别 | 保护要求 | 处理规则 |
|----------|----------|----------|
| 心理健康数据 | 敏感个人信息 | 单独授权 |
| 测评结果 | 高度敏感 | 加密存储 |
| 心理咨询记录 | 最高保护 | 限制访问 |
| 身份证信息 | 一般敏感 | 脱敏存储 |
